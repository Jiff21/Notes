Let's talk about CSS (H3 of 6 Header).

On this page we're going to talk about CSS. When writing CSS you can just style by using different element names on the page by using them as a selector. If you want to Select a Class, you need to put a period in front of the name .example . If you want to use an ID selector, use the Pound symbol in front of the name #example: Setting the overflow property to auto is a neat trick. When you are working with borders always remember TRBL (Top, Right, Bottom, Left). You can add multiple classes to a html element using whitespace. You make a Comment by using a backslash and a asterisk /* This is a comment */.

Selectors are very important in styling your page. Descendant Selectors just require a space, e.g. h2 p . Child selectors are similar but they don't target the whole page, e.g. h2 > p. Adjacent Selectors, e.g. h1 + p, H2 + p, can be useful if there if you want to select something that follows another thing directly and is part of the same parent, element. Attribute Selectors look for things with the thing in brackets e.g. a[title] or a[title="example"]. When using Attribute selectors you can use a tilde ~ before the equal sign to find anything with that will match as a full attribute values separated by whitespace or the asterisk * to do a wildcard search looking for literal values. If you use a carrot ^ to look for anything that starts with your search, while you can use a dollar sign ($) to look for anything that has that end. For more on selector and values please visit the W3C Page

Suedo-Class Selectors are used for selecting elements that are outside the DOM or in a state through User interaction. You use them by placing a colon, e.g. :hover before the word. The example is a Dynamic Selector which are the most common and can be used with a descendant selector, e.g. a:hover and whitespace does matter when doing this. When using the common linke ones be sure to remember Love Hate (link, visted, hover & active) so they behave correctly. Learn UI element state selectors when they become supported. Structural Suedo-Class Selectors can be used to target elements based on pattern regarding sibling element, e.g. :nth-child(odd). Structural Selectors use terms like first-child, only-child or last-of-type. You can also target things using before and after , e.g. a::after. Sometimes you may want to use the content property with the attribute to display the value of an attribute. a::after{content:" attr(href)";}.

Resolving conflicts is an important part of CSS. The Cascade is what you think it is and the last rule to get parsed will win... most of the time. Inheritence is also what you think it is, but it's important to remember that when you use relative properties these effects can become cumlative. Specifity is where it gets complex. ID tags are worth 100 Points, Classes are worth 10 Points & Elements are worth 1 (so "#aside p" would be worth 101 points while "div p"" would only be worth 2). The Important Declaration can overide all of this color:green !important;, but you shouldn't really use it because it's the cheap way out and will make code impossible to fix later.

Let's talk about font formatting. Remember font-family properties are case sensitive and fonts should be stacked in case they do not have it. You can use @font-face to use fonts the User may not have, if you do be sure to put it at top of your style sheet and order the styles (normal, italic, bold, bold-italic). text-transform can be used to change capitalization and cover up inconsistencies with the source text.

The property for aligning text is called text-align. Kerning doesn't exist yet but may be comiing with CSS3, for now we have to live with letter-spacing and word-spacing. Rendering matters a bit and text will look a bit better if you optimize for legibility text-rendering:optimizeLegibility or can load faster if you go for speed text-rendering:optimizeSpeed. It may seem simple but you can Indent text using the text-indent property text-indent: 3em. You can use a negative indent to create hanging punctuation if you have quotes that are in their own paragraph. If you're a graphic designer you know line-height as ledding. If you're going to let line-height be inherited you should use numerical values. line-height: 1.8. Remember Margin and Padding does not inherit unless you tell it to.

The Box Model is the rectangular box that all your elements are contained in and are made up of the Margin, Padding, Border, width and height. Padding is the space in between the content and the border. While the Margin is the space between the border and the edge of the box, although they shouldn't be counted on to set the container box because the can be ignored if there's not enough room. When calculating the size of a box it's important remember to double the border, padding, and margin space because if it's appearing on both sides of the element. Margins act funny vertically because of Vertical Margin Collapse and Floats. Vertical Margin Collapse is when two margins directly touch only the largest value is counted (borders and padding can prevent them from toucing directly). You can also use margin-left:auto with margin-right:auto to auto-cent objects with defined widths. With Padding the same sort of thing can happen horizontally if there is not enough space in the parent element. You can use border-radius with borders of unequel thickness to make some Cool Edges. If you're using a background image in an external CSS file the link is relative to the CSS file not the html. When using background-poistion the first number represents horizontal while the second number is the vertical. If you use percentages with background position it will find that spot on the image then center it at that percentage of the element. If you use multiple images be sure to stack the top one first and use a comma to seperate the url() selectors then use the background-position listing the positions respectively also seperated by a comma.

There are a few Basic Color Keywords that have been around since before time. There's also an Extended List that's pretty well supported as well. You can also use Hexadecimal Notation(#ab19ef) where each set of two characters represents red, green and blue respectively on a chart. If the pairs match you can use a 3 digit shorthand (#fff). RBG Values ( rgb(324, 235, 12)) are red, green, and blue values up to 255. If you site has a Color Palate you may want to leave the color values at the top of your style sheet in a comment so they can be used without memorization. Another option is HSL, it uses a 0-360 Hue value and percentages of Saturation and lightness (hsl(240,80%,40%)). Opacity can be set to a value between 0 and 1 in decimals. And while using that term is the only one that works on images, you can set element values using rgba or hsla the "a" being an alpha channel that also a 0-1 value.

Drop Shadows can be added with the box-shadow property ( box-shadow:10px 10px 5px 5px). The first two numbers are horizontal and vertical length respectively, the third is blur radius and the last is spread. Gradients are fairly new but they are rapidly being adopted. linear-gradient(top, #00000 0%, #ffffff 100%)
