QARK

Description

QARK (Quick Android Review Kit) is a tool that does static code analysis on Android App to locate possible security issues, it can examine source code or an APK. It then provides an HTML report of known vulnerabilities and can compile a QARK app that will allow you to test possible vulnerabilities on a device or emulator.

Install & Run

All you really need to do is clone QARK

git clone git clone https://github.com/linkedin/qark.git

cd into the folder, and run with the follind command.

python qark.py

You can simply follow the prompts to get a report and run tests against evil twin of the app.

Retire.js

Installation

Retire.JS is the javascript equivalent to Dependency Check, it does static code analysis and lists know vulnerabilities with components you are using. As long as you have NPM it's super simple just run the following command.

sudo npm install -g retire

Instructions

If you want to see vulnerabilities in the Terminal, simply run:

--retire --path ~/Path/To/Project/Root/

I prefer to read them in a text file, so I can separate issues and mark things off as I go. This command will write a file in your Documents folder with whatever name you choose, output format can be .txt or .xml.

retire --path ~/Path/To/Project/Root/To/Test/ --outputpath ~/Documents/AnyFileName.txt

Source Clear (scr:clr)

Intro

SourceClear (src:clr) is a static vulnerability scanning tool that can also scan active websites for components with known vulnerabilities. The scr:clr vulnerability library can searched manually without signing up. https://www.sourceclear.com/registry/explore

Install Steps.

Once logged in Install agents on your github repository using the side nav.

To install for local scanning:

brew update
brew tap srcclr/srcclr
brew install srcclr
srcclr activate

Use the activation code found in the Token dropdown in the Agents side nav.

To run a scan on a URL:

srcclr scan --url https://github.com/yourgithubname/repositoryname

To run a scan on a local repository:

srcclr scan ~/path/to/the/code/base

Tips

JIRA

In Team settings you can integrate with a JIRA project so you can select an issue an hit Create Issue to make linked JIRA issues. You may need pro for this.

Ignore

Say you find a vulnerability in a test scope or you decided it doesn't matter you can choose to ignore it by clicking on issues tab in the Web GUI, Click the Select Button, then click Ignore Button on the top right of the search bar menu bar. Then you will not get notified about the issue or have it clutter your report.

Notifications

If you click on your User Name in the top right and select Notification Settings, you can control your notification setting. These are only available on the Pro Plan but you can get an email every time a new Vulnerability is found. If you have weekly summaries turned on and a new vulnerability is found in an old scan you will get notified even if you don't perform a new scan on your code.

Vulnerable Methods

The Pro plan can scan python code for vulnerable methods.
