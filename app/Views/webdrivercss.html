Introduction

Webdrivercss is a visual regression testing tool. Its advantage over PhantomCss is that you can run it in actual browsers like Chrome, Firefox, Internet Explorer, & Safari. Note running tests in actual browsers will take considerably more time than using Phantomjs browser, that being said PhantomJS is sometimes like testing for a Browser that nobody actually uses. In my experience you get way more useful issues testing in the browser you support.

The bummer about WebdriverCSS is there in not support yet for WebdriverIO 3 or 4 yet. So if you wanted to do functional testing with Webdriverio and visual testing with Webdrivercss, you would either need to use the old webdriverio for functional tests (Which is confusing with all the new documentation) or get creative about using a Global install for the newer webdriver and a locale install for webdrivercss.

Install

Install steps can be found on their github page, but here is what worked for me in 2016 to add it to a project. Perquisites Homebrew, and NPM.

brew install graphicsmagick
cd path/to/your/project
npm install webdrivercss@1.1.10 --save-dev
npm install webdriverio@2.4.5 --save-dev
npm install wdio-mocha-framework@3.0.2 --save-dev
npm install chai@3.5.0 --save-dev

Running

With the above example files structure you could run a single test with:

node ./qa/webdriverio/specs/smoke/basic_tests.js

Or a suite with

node ./qa/webdriverio/specs/smoke/*.js

The first time you run it will simply just add baseline screenshots to the baseline-screens folder set in the config. Then on future runs it will compare the new photos to the baseline photos and look for mismatches, then highlight them in screenshots in failed-diff-screens folder. If intentionally change a page and need to rebase the screenshots so future runs test against the new page design, in the 'require('webdrivercss').init(client' section of the setup change updateBaseline to true (case sensitive) and run. Then change it back to false so future runs will compare against the new baselines.
