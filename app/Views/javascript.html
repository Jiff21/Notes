A few basic things about Javascript, it is case sensitive, but not whitespace sensitive. It's important to remember that the code will get executed when the browser hits the code. Best practice is to run the Javascript at the end of the body unless you're linking to a stylesheet. If you want you're variable to be a string just be sure to put it in quotes. You can quote inside it by using a different type of quote.

Variables are about as important as it gets, var foo; is an undefined variable, while var foo=100; is defined. Variables can be made of letters, numbers, underscores and the dollar sign, but can't start with a number. Some issues can up when you go to change a variable and you accidently don't get the name right because of case sensitivity, this will create a new variable and screw with your mind because an error doesn't appear. Javascript is a weekly typed language meaning you don't have to declare if variables are a string, number or boolean.

Conditionals Code is done with if statements. Here's an example:

if (x==100){
     alert(this)
}else{
     alert(that)
}

You can nest another argument inside the else, but it is suggested to not nest more than once or twice for the sake of code readiblity.

Another important part of javascript is Operators. Arithmetic Operators are things like addition, subration, multiplacation and division foo = foo + 1; or in shorthand foo += 1;. But it's important to remember Oparator Precedence, multiplication and division are considered more important than addition and subtraction, use parentheses to ensure your math gets done in the order you intended. For Equality rember that you need to use double equal signs foo == '5' or three equal signs for Strict Equality which also checks type foo === 5. When making Comparisons you have to remember that and uses a double ampersand && and for Or you use two bars ||. Modulus % will leave you the remainder. The Ternary operator is like a mini if statement, you make one by setting a condition then using a questionmark then do this is true colon this if false (e.g. (playerOne > playerTwo ? playerOne : playerTwo)).

Loops

A For-of loop is writen like:

for (var value of myArray) {
     console.log(value);
};

Working with Loops , also known as iteration, is a big part of using Javascript. A for loop starts with the word "for" then opens parenthesis, in the parenthesis you start with a variable being defined, after a comma you set a condition, after another comma you set and incrementor and then close the parenthesis. You follow with curly braces, that will get executed every time the loop runs.

A For loop is writen like:

for (var = i, i < 10, i++){
     console.log(i);
};

A while loop is writen like:

while (a < 10){
     console.log(a);
     a++;
};

You can always use break to manually stop a loop. You can also use continue to skip one iteration and check the condition again.

Another big part of Javascript is writting functions. A function is declared this:

function myName('n') {
     console.log(n);
};
const myName = (n) => {
     console.log(n);
};

Then you can use the function by calling it with parenthesis and then following it with a semicolon (e.g. myName();). Some functions have parameters that go inside the parenthesis. If you call a function with extra parameters it will just ignore the extras, but if you pass in too few they will be considered undefined. It's also important to remember Variabal Scope . If you define a variable in a function it is a local variable and will be undefined outside the function. If you need a global variable just declare the it outside the function and don't use the word var when defining it inside the function.

Arrays are like Variable for with multiple values. You define an array the same way you would a variable except you put the values in a square bracket separated by commas (e.g. var myArray = [0, 1, 2, Cat]; ). Remember that to start counting at 0 when referencing the items in the array. If you need to log out a simple array you can do this quickly with console.log(JSON.stringify(array));.

Objects are containers that hold things like variables or functions. You make a variable like this:

var player = new Object();
player.name = "Fred";
player.score = 100;

But this also has a shorthand (var player = {name: "fred", score = 100};). You can then make a method for the object by writting a function like this:

function playerDetails(){
     console.log(this.name + " has a score of " + this.score ".");
}

One of the most important methods of the Document Object Model is getElementById which can be used to retrieve an element node (var myElement = document.getElementById('someId')). You can use this to grab things in the DOM and change them with .setAttribute. More helpful is to individually drop element nodes into the DOM, using .createElement with .appendChild, for more on this please watch this video. You should also know than querySelector can be used on the document to grab elements by css.

Methods are used on a bunch of different objects. When using methods for dates it's important to remember that .getDate will give you the day of the month, while .getDay will give you the day of the week, and that to get the year you should use .getFullYear (e.g. "myDate.getFullYear(): ). .getHours will give you the hour in a 0-23 format. .getTime will give you the time down to the millisecond since 1/1/1970, which gets used a lot to see if something is newer than something else. When comparing dates you usually want to use a method because you're comparing objects and they're not same even if they have the same value (e.g. date1.getTime() == date2.getTime(); ).

Events are a huge part of interactivity. These can be done in dot notation and then written like a function ( document.onclick = function(){...). You can also do the by using an Event Listener which isn't completely supported because IE is IE (document.addEventListener('click', myFunction, false)). If you want to target something you can use a combo of a new variable and the .getElement function like this:

var myImage = document.getElementById('mainImage');
myImage.onclick = function(){
     alert('You click me');
};

You can use the the onsubmit function for forms to stop them from sending to the server by setting them to return false. If you're planning on adding CSS that will require Javascript to complete the function you may want to only add the CSS through a javascript, this type of practice is called Progressive Enhancement.

You can change the inlined CSS with the .style (.style.red ). Some of the terms change slightly, things that are hyphenated change to camel case (backgroundColor). Class is a reserved word so in Javascript you use className.

Regular Expressions can be used to do some complex stuff, check this video for more information. You're also going to want to learn more about Asynchronous JavaScript and XML, also known as AJAX to make the browser do all this stuff on the server side.

If you are trying to get a single value out of an associative array you could use the forEach function inside a for loop.

stocks.forEach(function(stock)){
     symbols.push(stock.symbols);
};

This could probably be done more efficiently with map. It's similar to forEach but returns a new array by default. The function run needs to take in the element run as an argumen & return the new element that’s going to take its place in the new array.

stocks.map(addSymbol(stock));

But if you are dealing with associtive arrays you can simplify this by using the map function.

function getStockSymbols(stocks){
     return stocks.map(function(stock){
          return stock.symbol;
     });
};

Of if you need to only get elements that pass a certain qualification you could use the filter method.

function getStocksOver(stocks, minPrice){
     return stocks.filter(function(stock){
          return stock.price >= minPrice;
     });
};

The advantage of using map and filter together instead of looping through data is it will work well with asynchronous data, while loops generally do not. The end result would look something like this.

var fileredStockSymbols = {
     stock.
          filter(function(stock){
               return stock.price >= 150;
          }).
          map(function(stock){
               return stock.symbol;
          }).
};
