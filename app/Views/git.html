Starting a Project

Locally use:

git init
touch README.md
git add *.c
git add README.md
git commit -m "This is the 1st commit"

Pull down a project that has already been started:

git clone github/url

Stash

Save current changes to Stash
git stash -m "Message about stash"

List stash changes
git stash list

Get a stashed change back to working directory:
git stash pop HashFragNum

Clear all stashed changes:
git stash clear "Message about stash"

Basics

Check for Changes since last commit:
git status

Add a file to local staging:
git add path/to/file

Move a file:
git mv path/to/filename.txt new/path/to/filenamenew.txt

Commmit all staged file:
git commit -m "Message with commit"

Stage & Commmit all changes in working directory:
git commit -am "Message with commit"

Push commits to remote:
git push

Get rid of an unstage change:
git checkout path/to/file/to/

Revert last commit (keeping changes) if not pushed to remote yet:
git reset HEAD~1

Revert last commit (nuking changes) if not pushed to remote yet:
git reset --hard HEAD~1

Full diff between two branches:
git --no-pager log branch1...branch2 --oneline

Branches

Create a new branch off current branch:
git checkout -b BranchName

Switch branch:
git checkout BranchName

Show Local branches:
git branch

Show Remote branches:
git branch -r

Put a local branch on remote with default tracking:
git push -u OriginName RemoteBranchName

Pull down a remote branch you don't have:
git checkout -b LocalBranchName origin/remote_branch_name

Delete local branch:
git branch -d BranchName

Merge a branch:
git merge NameOfBranchToMergeIntoCurrentBranch

Cherry pick a commit from another branch:
git cherry-pick HashOfChangeToMergeIn or git cherry-pick olderHash..toNewerHash

Git Settings

Your syncing your .bash_profile now and autocompletion can be setup by simply setting your bash profile to use the files in this project:
source Users/$USER/Documents/Code/Notes/app/examples/.bash_profile

You should be able to use your synced git ignore by configuring it to look at the file in this project
git config --global core.excludesfile Users/$USER/Documents/Code/Notes/app/examples/.gitignore_global

Git Bisect

Git has a nifty feature for tracking down where a bug started. Figure out a past commit that's working (good hash), then follow these steps.

git bisect start
git bisect bad
git bisect good HashFragNum

Keep repeating git bisect bad and good commands until landing on the commit that broke the code. Then type the following command and return to the head to fix the broken code.

git bisect reset
