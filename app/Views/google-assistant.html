Starting a Project

Create a directory and place gactions file in it. CD into it and run:

chmod +x gactions
mkdir new_project && cd new_project
gactions init
git add README.md
npm install express body-parser actions-on-google --save
touch app.js

Full instructions. Look into npm i actions-on-google --save-dev install

Previewing

To Use the online previewer for a Google Action

gactions preview --action_package mypackage.json

or you have these options avialable:

gactions preview --action_package mypackage.json --preview_mins 666 --invocation_name "Example Name"

It should also be noted that you need to say "Talk to Agent Name" to activate a foreign agent. For OK Google, do this we need to wait for Direct Actions

gactions simulate --debug

NGrok

NGrok is useful for quickly debugging:

brew cask install ngrok
ngrok http 8080

Use the http://90b5d2f2.ngrok.io address as your api endpoint in the action.json file> Then you should be able to run with:

gactions preview --action_package mypackage.json --preview_mins 666 --invocation_name "Example Name"
./gactions simulate
talk to Example Name

You can monitor ngrok connection at http://localhost:4040

Ngrok and cloud functions
  1. Design the dialogs, or user interface, for your action. Writing the dialogs upfront lets you catch mistakes, creates a natural sounding conversation, and gives you a good point of reference to code your actions when you're ready.
  2. Create an action package, A JSON manifest that describes your actions and defines other metadata.
  3. Define actions in the action package by doing the following:
    1. Define invocation and discovery patterns.
    2. Build dialogs and fulfillment.
  4. Test with the Web Simulator.
  5. Ensure you're ready for launch and take care of common issues.
  6. Publish the action and apply for an invocation name.
